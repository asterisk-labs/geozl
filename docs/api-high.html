high-level api

One call: compress().

Hand it a raster tile, get back an OpenZL frame. geozl picks the best graph for that tile — you don't touch the graph at all. The same call rumi builds on.

the whole thing 
frame = geozl.compress(tile)

a worked example

python — read a real DEM, compress it
# Copernicus DEM GLO-30, a 1°×1° tile over the Mont Blanc massif.
# Elevation in whole metres, so int16 holds it without loss.
import rasterio, numpy as np, geozl

url = ("https://copernicus-dem-30m.s3.amazonaws.com/"
       "Copernicus_DSM_COG_10_N45_00_E006_00_DEM/"
       "Copernicus_DSM_COG_10_N45_00_E006_00_DEM.tif")

with rasterio.open(url) as src:
    dem = src.read(1)                     # metres, float32

scene = np.round(dem).astype(np.int16)    # -> int16, lossless here

# Tile the scene and compress each tile on its own. Each tile
# gets the graph that wins it — that is where selection lives.
H, W = scene.shape
tiles  = [scene[r:r+512, c:c+512]
          for r in range(0, H, 512)
          for c in range(0, W, 512)]

frames = [geozl.compress(t, optim="balanced", lam=0.5)
          for t in tiles]

# Frames are self-describing: no options to decompress.
back = [geozl.decompress(f, dtype="int16") for f in frames]
assert all(np.array_equal(a, b) for a, b in zip(tiles, back))

Dataset: Copernicus DEM GLO-30, public COG on AWS Open Data (© ESA / Airbus / DLR). Numbers below are representative of one alpine tile, not a benchmark — see the Benchmark page for measured results.

how the call works

One call builds a small graph for the tile — predictor → zigzag → entropy backend — and searches for the graph that scores best on it. The predictor turns each pixel into a small residual against its neighbours; zigzag folds that signed residual to unsigned so the entropy stage packs it tightly; the entropy backend does the final coding.

Two things you set decide the search. method is a prior: it seeds the search from a predictor you expect to fit, but never pins it — if a different graph wins the tile, even one that drops the predictor and hands the bytes straight to zstd, that graph is kept. optim is the objective the search minimizes.

tile → predictor★ → zigzag → entropy★ → frame

★ chosen per tile by optim. store keeps the smallest frame, speed the fastest to decode, balanced minimizes bytes + lam · decode — where lam is the one knob that trades size against decode time.

parameters

tilendarrayA 2-D array, or 1-D with width given. Native byte order; 1, 2, 4 or 8 bytes per element.
methodstr · NonePredictor prior. planar (default), med, delta_w, delta_n, average, wp_static, none, or None to search every predictor unbiased.
optimstrObjective. store (default) smallest frame · speed fastest decode · balanced the trade-off.
lamfloatThe balanced knob: bytes + lam · decode. Required with optim="balanced", must be > 0; leave unset otherwise.
max_errorfloat · NoneNone or negative is lossless. A positive bound adds a lossy quant step that keeps every value within that error.
widthint · NoneRow width for a 1-D input. Read from tile.shape[-1] for a 2-D array.

Returns the frame as bytes. Raises ValueError for a bad argument and RuntimeError, carrying the encoder's own message, if compression fails.

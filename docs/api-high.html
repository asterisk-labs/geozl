high-level api

Measure, pick, run.

Three calls cover the whole surface. profile measures every candidate graph on your tile, compress runs the search and keeps the winner, decompress hands the array back.

Run in Colab Download the notebook

the three calls geozl 0.7.6

profile(tile)

Compresses and decompresses every graph in the grid and returns one row each. A benchmark you run to decide, not part of the compress path.

compress(tile)

Searches the same grid and keeps the graph that wins this tile. Returns the frame as bytes.

decompress(frame)

The array back. dtype and width say how to read the bytes, because the frame does not carry them.

Every number on this page comes from one run on the tile above, a 512×512 window of the Copernicus DEM GLO‑30 over the Mont Blanc massif, public on AWS Open Data (© ESA / Airbus / DLR). Elevation is in whole metres, so int16 holds it exactly, and smooth terrain is where spatial predictors shine. For results across datasets see the Benchmark page.

what a graph is

Every candidate has the same shape. An optional predictor, then zigzag to fold the signed residual to unsigned, then a backend that does the actual coding. The recipe string names the pieces in order, so planar>zigzag>transpose>entropy is one graph and id>store_lo is another.

tile → predictor → zigzag → layout → backend → frame

Predictors · method seeds the search from one

planar
Predicts each pixel from its neighbours as W + N − NW, the local gradient. Good default for smooth fields.
delta_wdelta_n
The plain difference from the west or the north neighbour.
med
The median predictor, JPEG‑LS style. Picks among W, N and W+N−NW by the local gradient. Best ratio on many tiles, slower to decode.
average
Predicts from the average of the neighbours.
wp_static
Fits a small weighted predictor to the tile and stores the weights in the frame header.
delta_1d
OpenZL's built‑in 1d integer delta, with no spatial notion at all.
id
No predictor. The raw stream goes straight to a backend, the escape hatch for a tile a predictor does not help.

Layout and backends · what closes the graph

entropy
Adaptive FSE and Huffman.
zstdfield_lz
LZ backends, which also catch the repetition that entropy coding alone misses.
transpose
Splits the int16 into byte lanes, the low byte and the high byte as separate streams, then codes them. This is the byte shuffle blosc and EOPF use, so id>transpose>zstd is exactly the EOPF baseline.
store_lo
Transposes, then stores the low lane raw because it is sensor noise and not worth coding, and entropy‑codes the high lanes. Wins on Sentinel‑2, where the low byte is irreducible.

profile measure the candidates

profile compresses and decompresses every graph in the grid and returns one row per graph. The timing runs in C with the checksum off, so the speed is the pure codec and not the Python call or frame verification.

method sets the prior. "planar" builds the grid from planar and id, None sweeps every predictor, "none" is the id branch alone. reps is how many times each graph is timed, and the fastest rep is kept.

shannon_pct is the frame size against the tile's order‑0 entropy. Over 100 means the graph beat that entropy by finding structure the byte histogram misses, which every spatial predictor here does by a wide margin.

compress run it for real

compress searches the grid and keeps the best graph for the tile. optim sets what best means. store minimises the frame size, speed minimises decode cost with size breaking ties, balanced minimises a weighted mix of the two.

The selector scores each candidate and keeps the argmin, with b the measured compressed bytes and d the modelled decode cost. Normalising by the per‑tile minimum makes lam a pure weight that means the same across tiles of different size, so lam=0 is store and a large lam is speed.

a prior, not a pin method seeds the search from a predictor you expect to fit, but it never fixes the outcome. If a different graph wins the tile, even one that drops the predictor entirely and hands the bytes straight to zstd, that graph is the one kept.
storebi speeddi balancedbimin b + λ · dimin d
python · return_graph gives back the recipe that won
for optim, kw in [("store", {}), ("speed", {}), ("balanced", {"lam": 0.5})]:
    frame, recipe = geozl.compress(tile, optim=optim, return_graph=True, **kw)
    print(f"{optim:9} {raw/len(frame):5.2f}x  {recipe}")
store 4.50x planar>zigzag>transpose>entropy speed 1.55x id>store_lo balanced 4.49x planar>zigzag>entropy

Once profile has told you the winner you can skip the search altogether, since graph= takes a recipe string straight from the table.

python · pin the recipe, no search
best  = rows[0]['graph']
frame = geozl.compress(tile, graph=best)
planar>zigzag>transpose>entropy 4.50x

max_error bounded‑error lossy

By default compress is lossless, and so is any max_error of 0 or less. Pass a positive max_error and it prepends a quantisation step that trades exactness for ratio with a hard guarantee. No reconstructed value moves by more than max_error.

The quantiser is uniform, and the rounding to the nearest multiple of the step is what costs at most half a step. What travels in the frame depends on the type. On integers the encoder stores the reconstruction and the decoder copies it. On floats it stores the index, because quantising a float is what turns it into the small integers the downstream predictors compress, and the decoder multiplies back. Both forms reconstruct the same values.

two caveats for integers The step is floored at 1 and truncated to a whole number, since an index cannot be finer than the sample. So max_error=1.2 gives a step of 2 and behaves exactly like max_error=1, and anything below 0.5 gives a step of 1, which is an exact round trip. That is why the curve is a staircase and not a smooth slope.
stepΔ = 2 · max_error indexq = round(x / Δ) reconstructionx̂ = q · Δ guarantee|x − x̂| ≤ Δ / 2

decompress get the array back

geozl works one level below the array. A tile enters OpenZL as a numeric stream described by one thing, the bytes per element. An int16 and a uint16 are both two bytes to the frame, which stores the exact bytes but not the sign, and not that they were 512×512 rather than a flat vector.

So on the way out you say how to read the bytes. dtype reinterprets each element, and leaving it off gives an unsigned view of the right width. width reshapes to 2d, and leaving it off gives a flat 1d array.

This is deliberate. geozl is the fast primitive that carries no metadata it does not need to compress. The layer above it, rumi, owns the dataset, wraps each frame with the real dtype and shape, and decompresses with no arguments at all.

python
back = geozl.decompress(
    frame, dtype="int16", width=512
)
(512, 512) int16

the loop measure, pick, run

python
rows  = geozl.profile(tile)                              # see what wins
best  = rows[0]['graph']                                # take the top recipe
frame = geozl.compress(tile, graph=best, max_error=4)   # pin it, no search
back  = geozl.decompress(frame, dtype='int16', width=512)
planar>zigzag>transpose>entropy 9.93x lossy 0.05 MB max error 4.00

Ten times smaller, and every value within 4 m of where it started. The error panel is stretched to ±5 m so there is anything to see at all. What structure is left follows the valley floor and the ridge lines, which is exactly where the terrain stops being locally planar.

parameters

compress(tile, ...) → bytes

tilendarrayA 2‑D array, or 1‑D with width given. Native byte order, 1, 2, 4 or 8 bytes per element.
methodstr · NonePredictor prior. planar (default), med, delta_w, delta_n, average, wp_static, none, or None to search every predictor unbiased.
optimstrObjective. store (default) smallest frame · speed fastest decode · balanced the trade‑off.
lamfloatThe balanced weight. Required with optim="balanced", must be > 0, leave unset otherwise.
graphstr · NonePins a recipe and skips the search entirely. Takes a string straight from the profile table, such as planar>zigzag>transpose>entropy.
max_errorfloat · NoneNone or anything ≤ 0 is lossless. A positive bound adds the quant step that keeps every value within that error.
widthint · NoneRow width for a 1‑D input. Read from tile.shape[-1] for a 2‑D array.
return_graphboolReturns (frame, recipe) instead of just the frame, so you can see which graph won.

Raises ValueError for a bad argument and RuntimeError, carrying the encoder's own message, if compression fails.

profile(tile, ...) → list[dict]

methodstr · NoneThe same prior as above. Here it decides which grid gets built and measured.
repsintHow many times each graph is compressed and decompressed to time it. The fastest rep is kept.
max_errorfloatProfiles the lossy path, so the whole grid is measured with the quant step in front.
→ rowdictgraph, ratio, encode_mbps, decode_mbps, shannon_pct. Sorted by ratio, best first.

decompress(frame, ...) → ndarray

dtypestr · NoneReinterprets each element, so "int16" reads the 2‑byte elements as signed. Left off, you get an unsigned view of the right width.
widthint · NoneReshapes to 2d. Left off, you get a flat 1‑D array.
